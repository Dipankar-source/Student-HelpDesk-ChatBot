Firebase Firestore Rules Setup

Resolve "Missing or insufficient permissions" errors

Problem

Your application is receiving "Missing or insufficient permissions" errors because your Firestore security rules are too restrictive.

Warning: The rules below will allow full read/write access to your database. Use them only for development and testing.

Solution

Copy and paste the following rules into your Firebase Firestore Rules editor:

Firestore Security Rules
Development Mode
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    // Allow read/write access to all users under any conditions
    // Warning: These rules are for development only!
    match /{document=**} {
      allow read, write: if true;
    }
  }
}

How to Implement

1

Go to the Firebase Console and select your project.

2

Click on "Firestore Database" in the left sidebar.

3

Select the "Rules" tab.

4

Replace the existing rules with the rules provided above.

5

Click "Publish" to save your new rules.

Go to Firebase Console

For Production

Once you've verified your application works correctly, you should implement more secure rules:

Firestore Security Rules
Production Mode
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    // Allow users to read/write their own data only
    match /sessions/{sessionId} {
      allow read, write: if request.auth != null && request.auth.uid == resource.data.userId;
      allow create: if request.auth != null && request.auth.uid == request.data.userId;
    }
    
    match /messages/{messageId} {
      allow read, write: if request.auth != null && request.auth.uid == resource.data.userId;
      allow create: if request.auth != null && request.auth.uid == request.data.userId;
    }
  }
}
Note: These production rules ensure that users can only access their own data, providing much better security for your application.
